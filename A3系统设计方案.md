# A3-基于大模型的个性化资源生成与学习多智能体系统设计方案

> 中国软件杯 第十五届 A3赛题 | 出题企业：科大讯飞
> 设计方向：Web应用

---

## 一、目标用户定义

### 1.1 典型用户画像

#### 学生端

| 画像维度 | 大学生（主目标） | 研究生 | 职场学习者 |
|---------|----------------|--------|-----------|
| **典型人群** | 大二~大四计算机/电子/AI相关专业 | 研一~研三，有明确研究方向 | 1-5年经验的IT从业者，想转AI/提升技能 |
| **使用场景** | 期末备考、课程预习、实验课辅助 | 论文阅读、研究方法学习、前沿知识补充 | 碎片化学习、技能进阶、项目驱动学习 |
| **技术基础** | 有编程基础，对AI有基本认知 | 有一定科研能力，需要深度内容 | 实践经验丰富，需要体系化梳理 |
| **学习习惯** | 集中式学习，考试驱动 | 探究式学习，论文驱动 | 碎片+周末集中，解决问题驱动 |
| **薄弱环节** | 知识点衔接不清、缺乏实践指导 | 前沿知识获取效率低、论文理解困难 | 理论体系不系统、新技术落地难 |
| **期望价值** | 精准复习资料、针对性练习 | 个性化论文导读、研究方法推荐 | 技能图谱+实践项目推荐 |

#### 教师端（辅助角色）

| 画像维度 | 高校教师 |
|---------|----------|
| **典型人群** | 计算机/电子信息类课程授课教师、实验课导师 |
| **使用场景** | 备课辅助、学情分析、个性化作业布置 |
| **核心痛点** | 无法兼顾每个学生的进度，作业/试卷出题耗时，对学情掌握有限 |
| **期望价值** | 自动化出题、学情数据分析、课程资源智能生成 |

### 1.2 核心痛点分析

| # | 痛点 | 描述 | 影响 |
|---|------|------|------|
| P1 | 学习资源爆炸但匹配难 | 网上课程/资料海量，学生不知从何学起，学了不匹配的内容效率极低 | 学习路径混乱，挫败感强 |
| P2 | 千人一面的教学模式 | 传统课堂同一进度、同一内容，基础好的"吃不饱"、基础差的"跟不上" | 两极分化严重 |
| P3 | 缺乏即时个性化反馈 | 作业一周后才知道对错，学习过程中的问题无法实时得到针对性解答 | 错误认知固化成习惯 |
| P4 | 学习资源形式单一 | 大多只有文字/PPT，缺乏视频、交互式练习、代码实操等多模态形式 | 学习效果差，容易疲劳 |
| P5 | 学情评估粗放 | 仅靠考试分数评价，无法细粒度了解知识薄弱点 | 复习无的放矢 |
| P6 | 教师出题/备课负担重 | 教师要花大量时间出题、准备教案，无法专注于个性化辅导 | 时间和精力瓶颈 |

### 2.2 智能体详细设计

#### Agent 1：诊断Agent —— 学习诊断师

| 项目 | 内容 |
|------|------|
| **角色定位** | 学生的"私人学情分析师" |
| **核心能力** | 多轮对话式评估、知识图谱推理、薄弱点定位 |
| **输入** | 用户自然语言对话、答题结果、学习历史 |
| **输出** | 6维动态学生画像（JSON） |

**画像维度（≥6维）：**

| 维度 | 描述 | 获取方式 |
|------|------|---------|
| ① 知识基础 | 用户目前掌握的知识范围和深度（如：Python基础 ✅ / 机器学习 ❌） | 对话+初始测验 |
| ② 认知风格 | 偏好阅读型/动手型/视听型/混合型 | 对话引导判断 |
| ③ 易错点/薄弱点 | 具体到知识点的薄弱环节（如：线性代数-特征值分解不理解） | 诊断问答+错题分析 |
| ④ 学习目标 | 短期目标（如：完成机器学习课程）和长期目标（如：找到AI相关工作） | 对话采集 |
| ⑤ 学习节奏 | 每天可投入时间、专注力时长、偏好的学习时段 | 对话+行为跟踪 |
| ⑥ 专业背景 | 所在专业、年级、已修课程、相关项目经验 | 对话+对接教务 |
| ⑦ 兴趣偏好（扩展） | 对哪些子领域更感兴趣（如：CV > NLP > 强化学习） | 对话挖掘 |

**诊断流程：**

```
用户进入 → 引导式对话（设计5-8轮） → 穿插微型测验（3-5题） 
→ 大模型推理分析 → 输出结构化画像 → 画像入库（并同步给其他Agent）
```

#### Agent 2：规划Agent —— 学习路径规划师

| 项目 | 内容 |
|------|------|
| **角色定位** | 学生的"私人课程顾问" |
| **核心能力** | 知识图谱导航、拓扑排序、自适应规划调整 |
| **输入** | 诊断Agent输出的学生画像 + 课程知识图谱 |
| **输出** | 个性化学习路径（步骤序列+时间节点） |

**规划策略：**

| 策略 | 说明 |
|------|------|
| **目标分解** | 将学习目标拆解为可执行的子目标（如：学完线性代数→学完ML基础→完成项目A） |
| **前置依赖** | 识别前置知识点，确保学习顺序科学性（先学A再学B，而不是跳跃） |
| **难度渐进** | 内容难度从"舒适区边缘"开始，逐步递增（Vygotsky最近发展区理论） |
| **时间分配** | 根据用户每日可用时间，将学习任务切分为合理粒度（每任务30-45分钟） |
| **动态调整** | 学习过程中根据完成情况、测验分数实时调整后续路径 |

**输出示例：**

```json
{
  "learning_path": [
    {
      "step": 1,
      "topic": "Python基础回顾",
      "estimated_time": "120min",
      "resource_types": ["文档", "代码练习"],
      "prerequisites": [],
      "difficulty": "easy"
    },
    {
      "step": 2,
      "topic": "线性代数核心概念",
      "estimated_time": "180min",
      "resource_types": ["视频讲解", "练习题"],
      "prerequisites": ["step_1"],
      "difficulty": "medium"
    }
    // ...
  ],
  "total_estimated_time": "40h",
  "milestones": ["第1周-基础巩固", "第3周-入门完成", "第6周-项目实战"]
}
```

#### Agent 3：生成Agent —— 内容创作师

| 项目 | 内容 |
|------|------|
| **角色定位** | 学生的"私人教材编写团队" |
| **核心能力** | 多模态内容生成、自适应难度调整、风格匹配 |
| **输入** | 规划Agent的学习路径 + 画像中的认知风格 |
| **输出** | 5种以上个性化学习资源 |

**资源生成类型（≥5种）：**

| # | 资源类型 | 说明 | 生成策略 |
|---|---------|------|---------|
| 1 | **专业课程讲解文档** | 针对当前知识点的讲义，含概念讲解、公式推导、案例 | 根据用户认知风格选择讲解方式（类比/公式/代码示例） |
| 2 | **知识点思维导图** | 可视化呈现知识结构、概念关系 | 用Mermaid或Markmap生成，可交互缩放 |
| 3 | **练习题目** | 选择题、填空题、编程题 | 根据画像中"易错点"针对性出题，难度渐进 |
| 4 | **拓展阅读材料** | 相关论文、博客、技术文档推荐 | RAG检索+LLM摘要，匹配用户当前水平 |
| 5 | **多模态教学视频/动画** | 关键概念的动画演示 | 用Manim/JS动画生成，结合语音合成讲解 |
| 6 | **代码实操案例** | 带讲解的实战代码，含注释和运行说明 | 根据用户知识水平生成不同复杂度的代码示例 |

#### Agent 4：质检Agent —— 内容品控官

| 项目 | 内容 |
|------|------|
| **角色定位** | 内容的"终审编辑" |
| **核心能力** | 事实性校验、难度评级、格式规范检查、安全过滤 |
| **输入** | 生成Agent产出的所有资源 |
| **输出** | 质检报告（通过/修改建议/拒绝） |

**质检维度：**

| 维度 | 检查项 | 阈值 |
|------|--------|------|
| ✅ **准确性** | 事实性错误、公式错误、代码是否能运行 | 零容忍事实错误 |
| ✅ **难度匹配** | 内容难度是否落在用户"最近发展区" | 偏差≤1个难度等级 |
| ✅ **完整性** | 是否覆盖了规划Agent指定的所有知识点 | 覆盖率≥90% |
| ✅ **安全性** | 是否包含敏感内容、偏见、不当言论 | 零容忍 |
| ✅ **格式规范** | Markdown格式、代码高亮、链接有效 | 格式正确率≥95% |
| ✅ **可读性** | 句子复杂度、术语密度是否匹配用户水平 | Flesch可读性评分适配 |

**质检流程：**

```
生成Agent产出 → 质检Agent并行检查各维度 
→ 发现问题 → 反馈给生成Agent迭代修改 
→ 最多3轮迭代 → 通过后交付给前端
```

#### Agent 5：辅导Agent（可选加分项）—— 答疑助教

| 项目 | 内容 |
|------|------|
| **角色定位** | 7×24小时的"私人在线助教" |
| **核心能力** | 多模态答疑、上下文感知、Socratic引导 |
| **输入** | 用户提问 + 当前学习上下文 + 画像 |
| **输出** | 文字解答/图解/短视频讲解 |

**解答策略：**

| 学生类型 | 解答策略 | 示例 |
|---------|---------|------|
| 基础薄弱 | Socratic追问式：先引导思考，再给答案 | "你觉得梯度下降和最小二乘法有什么关系？" |
| 中等水平 | 类比+图解：用已知概念解释未知 | "反向传播就像你在玩'传话游戏'..." |
| 高水平 | 深度拓展：给出延伸阅读和对比分析 | "这个问题可以延伸到反向传播的数学推导..." |

### 2.3 智能体协同机制

```
┌─────────────────────────────────────────────────────┐
│                 Orchestrator（编排器）                │
│                                                     │
│  用户请求 → 解析意图 → 路由到对应Agent → 收集结果     │
│  → 若需要协同，触发Agent间通信 → 聚合输出             │
└─────────────────────────────────────────────────────┘
```

**典型协同场景 —— 生成一份个性化学习资料：**

```
Step 1: 诊断Agent → 获取用户画像
Step 2: 规划Agent → 基于画像确定"当前该学什么"
Step 3: 生成Agent → 根据规划产出讲稿+习题+思维导图
Step 4: 质检Agent → 检查质量，发现难度偏高 → 反馈
Step 5: 生成Agent → 调整难度重新生成
Step 6: 质检Agent → 通过
Step 7: 辅导Agent → 检查关联的常见Q&A是否已准备
Step 8: 编排器 → 聚合所有结果呈现给用户
```

## 五、关键技术选型建议

| 技术层面 | 推荐方案 | 说明 |
|---------|---------|------|
| **大模型** | 讯飞星火大模型 API | 赛题要求选用科大讯飞工具 |
| **多智能体框架** | LangChain / CrewAI / 自建编排 | 灵活选择，赛题未做限制 |
| **前端** | React + Next.js / Vue3 + Nuxt | 流式渲染流式输出、Markdown渲染、卡片化展示 |
| **后端** | Python FastAPI | 异步支持好，适合LLM调用 |
| **数据库** | PostgreSQL + pgvector / SQLite + Chroma | 向量数据库用于RAG |
| **RAG框架** | LangChain + ChromaDB | 构建课程知识库检索 |
| **思维导图** | Mermaid.js / Markmap | 前端渲染 |
| **动画生成** | Manim / Three.js | 教学动画可视化 |
| **代码编辑器** | Monaco Editor (VS Code Web版) | 在线代码编写与运行 |
